# FLOOR CAMPAIGN (2026-07-07, user strategy): every critical red becomes its weakest-form conjecture, then survives or dies

## The strategy (user-set)

For each of the 7 critical reds: (1) derive the WEAKEST FORM — the exact
statement its consumer(s) actually consume, pinned from the consumer's
packet arithmetic, no route structure; (2) re-pose the node as that floor
conjecture, standing until refuted; (3) PRE-REGISTER the falsifier before
collecting data (what counts, what does not — proxy/route failures never
count); (4) attack aggressively: numerics (Modal) + Pro briefs; (5) a node
that DIES forces early downstream re-thinking (better now than after
months of corridor work on a false region); (6) survivors — hard
conjectures resisting both proof and falsification — are THE TRUE
FRONTIER.

Template (established at dli, REPOSE_B_WEAK.md): pin the consumption from
the consumer's packet verbatim → weakest joint/exceptional form → evidence
ledger opens with the node's existing survival record → pre-registered
refutation standard with scaled-toy transport rules → routes to the
evidence lane; promotion back to conditional only after surviving
adversarial rounds (F-round rule).

## Program discipline

- One node at a time; each re-pose banked with its own REPOSE note +
  pre-registration + dag statement + ritual before the next begins.
- Attack phase per node: at least one direct numeric experiment (Modal)
  aimed at the floor's most refutable projection + a Pro brief when the
  node's object suits Pro's strengths.
- Survival ledger per node in the dag statement (+1 per absorbed attack).
- A death is a CAMPAIGN EVENT: stop, write the downstream-consequence
  analysis (which ambers/chains above are invalidated, what re-routes),
  and report to the user before continuing.

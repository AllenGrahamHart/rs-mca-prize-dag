# Upstream crosswalk - official checkpoint characteristic atlas

This is an exact finite-row preprocessor for upstream `(Q)` and split-pencil
work. Any packet involving a characteristic checkpoint should identify its
`(n,p,ord_n(p),floor(n/p))` atlas row before posing a theorem or compute
request.

The routing is mandatory: `m=1` has no two-fiber endpoint; `m=2` uses the
complement-square compiler; only the 16 `m>=3` pairs justify the general
higher-multiplicity eliminant. A PR should vendor the atlas verifier and table
as theorem evidence. It should not present the table as evidence for
flatness, a split-pencil census, or the final finite ledger.

The four `m=2`, remainder-two tuples have exactly `n/2` antipodal
minimum-width pairs. The six broad `m=2` rows are theorem-empty at that
width. All ten should be reported as analytically classified preprocessing,
not offered as compute requests.

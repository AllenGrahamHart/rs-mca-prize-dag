# Upstream crosswalk - Frobenius-checkpoint Q router

## `(Q)` interface

Przemek's first-form `(Q)` is an elementary-symmetric prefix, while the
Fourier/Myerson routes naturally use power sums. At depths reaching the
characteristic, the correct exact interface is not a cycle-error term: it is
the p-free power-sum ladder conditioned on `E_p,E_(2p),...`.

The official generated-field cap limits that list to at most 23 checkpoints.
A portable `(Q)` theorem may therefore prove a max-fiber estimate directly
for the coarse p-free fiber, which contains every conditioned mixed fiber and
incurs no checkpoint loss. Alternatively it may prove a max-fiber estimate
uniformly over checkpoint values and print how the Pade owner coalesces those
values. Summing over all `q^r` values costs `<n^453` and so preserves a
qualitative polynomial theorem, but it cannot pay the finite prize ledger.

## F2 relation

The terminology matches the p-free ladder and per-condition extras in the F2
campaign, but this node does not claim that the F2 growing-order Myerson
statement is uniform over arbitrary L1 received-word Pade graphs. A future PR
should present the 23-checkpoint interface as a typed consumer and keep the
missing conditional flatness theorem explicit.

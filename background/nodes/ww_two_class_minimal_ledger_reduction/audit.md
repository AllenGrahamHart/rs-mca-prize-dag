# Audit

The previous W3 interface required formulas and ownership for every named paid
structural class before defining its residual budget. That is stronger than
the consumer needs and can lose allowance when a paid upper bound is loose.

Paying only the exact planted set and defining every other list word as the
residual gives the exact identity `List=Plant disjoint_union NonPlant`. The
remaining W3 theorem is one direct upper bound on `NonPlant`; structural
columns may still be used internally to prove it, but are not interface
requirements.

This repair also removes generated-field normalization from the binding count.
The actual list is counted over the supplied row field. A proof route using
descent or generated subfields must justify that route, but the theorem's
quantity itself is unambiguous.

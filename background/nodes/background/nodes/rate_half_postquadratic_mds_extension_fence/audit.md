# Audit

The parity-check columns of the unweighted Reed-Solomon code include nonzero
dual multipliers, but those multipliers do not change their one-dimensional
spans. The proof uses only projective column directions.

Column farness is checked in syndrome space against a common one-column span,
not by asking whether each component is individually far. The example happens
to satisfy the stronger individual statement as well. The slope `0` is not
close; the four bad slopes are exactly `1,2,3,4`.

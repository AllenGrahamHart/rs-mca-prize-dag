# Audit - PMA sigma-one B11 scope

1. The theorem uses the exact `sigma=1` track only. It does not import a
   small-reserve conclusion into the asymptotic L1 lower-cutoff track.
2. `G_2<=2` uses touched petals: their support sizes are one or two. The
   proved maximal-sunflower tradeoff supplies at least two touched petals.
3. The first-match order is load-bearing. `GROW` is tested before `J/A2`, so
   `J/A2` here contain only `d<=2` members.
4. The finite failure concerns the printed B11 upper certificate. An upper
   bound exceeding an allowance is not a counterexample to the underlying
   class bound.
5. The comparison grants B11 the whole `n^6` primitive allocation. Any true
   post-full-petal residual allowance is no larger, so the route-cut direction
   is safe without inventing a mixed-branch budget split.
6. The `q>=n+1` deduction uses the generated field that actually contains the
   cyclic domain. It must not be replaced by an unrelated base or line field.

Mutation controls in `verify.py` reject `V_2=1`, an omitted Johnson
denominator, an inexact fourth-root boundary, and the claim that the B11
certificate fits at the two low rates when `q=n+1`.

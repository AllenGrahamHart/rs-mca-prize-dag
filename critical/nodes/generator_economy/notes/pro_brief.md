# Pro brief F — generator economy (cyclotomic-unit design)

*Self-contained; plays to classical cyclotomic-unit number theory. Construction
or obstruction both valuable.*

## Setup
`F_q`, `q ~ 2^256` prime; `mu_N' = <zeta> subset F_q^*`, `zeta` a primitive
`N'`-th root of unity, `N'` a 2-power quotient order (relevant range roughly
`80 < N' < 512`; open cells `N' in {128, 256}`). Fix a subset size `l' ~ N'/2`.
For an `l'`-subset `B subset mu_N'` write `e_1(B) = sum_{y in B} y` (the first
elementary symmetric / subset sum). A "certificate" for a family of `B`'s is a
poly-size set of base elements `g_1,...,g_G in F_q^*` such that every relevant
pairwise difference `e_1(B) - e_1(B')` lies in the multiplicative semigroup they
generate — because a NORM check `Norm_{F_q/F_p}(.)` is multiplicative, so
certifying the `G` bases certifies the whole (exponentially large) family at
once.

## The proved germ (single roots)
For the FULL difference set of `mu_N'` itself, differences factor as

```
    zeta^a - zeta^b = zeta^b ( zeta^{a-b} - 1 ),
```

so every root difference lies in the semigroup generated by
`G = N'` bases `{ zeta^k - 1 : k = 1..N'-1 } cup { zeta }`. Divisibility
distributes over products, so `N'` norm checks certify the entire root
difference set. This is the classical cyclotomic-unit structure (`(zeta^k - 1)`
generate the cyclotomic units up to roots of unity).

## The open problem
Lift the factorization from single-root differences to `l'`-SUMS. Concretely:

> Construct a family `F` of `l'`-subsets `B subset mu_N'` of size
> `|F| ~ B* / 2^33` (`B*` = the target cluster budget; `|F|` polynomial-in-`N'`
> below the fiber scale suffices for a proportional win) such that all pairwise
> differences `{ e_1(B) - e_1(B') : B, B' in F }` lie in the multiplicative
> semigroup generated by `<= poly(N')` base elements — so a poly-size norm
> check certifies the whole family.

**Admissibility guard (must hold):** the existence proof must NOT consume fiber
counts of the prefix map `Phi_sigma(A) = (e_1(A), ..., e_sigma(A))` — otherwise
it is circular with the `perfiber` estimate the design is meant to feed. Keep
the construction purely multiplicative/cyclotomic.

## The ask (choose your target)
- **(A) Construction:** exhibit such a family (e.g. `B` ranging over cosets /
  arithmetic-progression supports / sub-semigroup-structured supports, where
  `e_1(B)` itself factors as a bounded product of `(zeta^k - 1)` units), with a
  proof of the size and the `poly(N')`-base certification, respecting the
  admissibility guard.
- **(B) Obstruction:** prove no such economical family exists at
  `N' in {128,256}` (subset sums are too `additively` unstructured to factor
  multiplicatively over a small base) — a clean impossibility, equally useful
  as it closes this route.
- **(C) Partial design:** the best achievable trade `|F|` vs base-count `G` at
  small `N'` (a toy design search is a valid proportional win); a family of
  size `|F|` with `G` bases gives a certified sub-family of that size.

Anchor: at `N' = 16` (`F_17`, `zeta = 3`) a toy search should reveal whether
`l'`-sum differences factor over `O(N')` cyclotomic bases or blow up. The
number-theoretic heart is whether `sum_{y in B} y` inherits any of the
`(zeta^k - 1)` multiplicative structure for structured supports `B`.
